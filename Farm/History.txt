앱 시작시 로딩페이지를 만들고싶음.

	마지막 저장된(갱신된) 유기농업자재 데이터를 읽어온다.

)

보통 로딩작업을 각 플랫폼별로 하던데 공통으로 하고싶어서 OganicInput 프로젝트에서 하는걸로 결정했다.  

App -> AppShell -> Views 로 들어와서 로딩페이지를 만들면 로딩페이지에서 메인페이지로 이동할때 뒤로가기 같은게 자동으로 붙어버린다.

그러므로 로딩페이지는

App -> Loading -> App-> AppShell -> Views  구조로 만들었다.

만들고 앱 처음실행시 앱데이터로드할때 흰색빈화면이 출력되고나서(OganicInput.Android 실행되는 화면) 로딩페이지가 보이게 된다.

그래서 그 흰색화면 대체할 SplashActivity를 MS 에서 다운받아서 적용했다.
그러고보니 여기서 로딩작업하면 굳이 로딩페이지를 만들필요가 없는데.. 고민..

그래서 폰에 있는 다른 앱들을 봤다.

공통 : 보통 아이콘 색과 동일한 배경으로 처음 실행시키면서 가운데 로고가 보이던가, 텍스트가 보여지게 만든후에
1. 메인페이지로 이동후 데이터로딩작업
2. 같이 로딩작업

두가지 분류로 나눠졌다. 

메인 페이지로 이동후에 데이터 로딩작업같은 경우에 따로 페이지가 있지 않았다. 
메인 페이지 배경내에 스플래시 이미지를 띄우던가.
반투명회색으로 덮어씌우고 스플래시를 띄우던가 둘중 하나이다. 

Unity로 개발할때는 필요 없었던 기능이다. 
생각해보니 게임은 화면이 전체적으로 항상 다 바뀌면서 데이터로딩작업도 길고해서 화려하게 로딩페이지를 꾸미는데,
일반 앱의 경우에는 기본 테마를 자연스럽게 이어가야 하기 때문에 첫 시작 색부터 변경해야하는 것같다. 
또한 일반앱의 경우 1초 빈화면 그후 앱뜨는건 느리게 느껴진다. 게임은 원래 느릴꺼라 생각하니.. 뭐 유니티도 시작하자마자 로고 띄우니 다 같은 이유라 생각한다. 

결과적으로는 로딩페이지를 없애고, 
메인페이지로 들어온 순간 스플래시를 띄우고, 한번만 데이터를 조회해 주는걸로 결정했다.
